## 3. First Failure Analysis (Lesson 3)

### A. PDF Extraction / Layout (Gate A)
- **Exact Command**: 
  ```powershell
  python -m score2gp.cli extract-tab "fixtures/private/Lesson-3.pdf" --out "work/major_triads_benchmark_20260527_1552/lesson_3"
  ```
- **Observed Metrics**:
  - Page Count: 2 pages
  - Detected Systems: 49
  - Detected Bar Boxes: 61
  - Total text candidates: 594
  - Playable fret candidates: 546
  - Candidates with system: 449
  - Candidates with bar: 380
  - Fret candidates with system/string: 440
  - Fret candidates with bar: 379
- **Blocker Reason**: On Page 1, System 7, the vertical barlines are partially rejected. System 7 has exactly one right boundary accepted and a left boundary rejected, causing the edge boundary fallback to consider a left-edge inference. The fallback was rejected (`pdf_bar_box_edge_boundary_fallback_rejected`) because of the ambiguous left boundary candidate. This left 6 candidates unassigned to a bar box, resulting in a `partial` grouping status.

### B. Strict Build-IR (Gate B)
- **Exact Command**:
  ```powershell
  python -m score2gp.cli build-ir --musicxml "work/major_triads_benchmark_20260527_1552/lesson_3/omr/Lesson-3.mxl" --tabraw "work/major_triads_benchmark_20260527_1552/lesson_3/tab_raw.json" --out "work/major_triads_benchmark_20260527_1552/lesson_3/score.ir.json" --diagnostics-out "work/major_triads_benchmark_20260527_1552/lesson_3/diagnostics.json"
  ```
- **Blocker Reason**:
  1. The layout grouping status was `partial`, raising a strict geometry block.
  2. The OMR-generated MusicXML companion was rejected under the preflight timing check due to 74 underfull measures, flagging `musicxml_timing_risk`.
- **tooling/Benchmark Defect**:
  - The OMR timing anomalies are a known tooling limitation of Audiveris when parsing multi-page guitar tab layouts. 
  - To achieve a true semantic round-trip on Rung 3, a **verified, hand-crafted or properly engraved clean MusicXML companion** is needed for Lesson 3. Using noisy OMR-derived MXL files will continue to block the timing preflight gate.

## 4. Comparison Across Lessons

- **Lessons 4, 5, and 7** fail identically to Lesson 3 at the layout boundary, triggering `pdf_bar_box_edge_boundary_fallback_rejected` on their outer systems where barlines are partially ambiguous or incomplete.
- **Lesson 6** suffers a total system segmentation collapse. Only 3 systems were detected across a PDF containing 1125 fret candidates. The lines are fragmented (`pdf_tab_staff_lines_fragmented`), meaning the staff geometry could not be resolved. This is a severe product layout heuristic defect.

## 5. Recommended Next Implementation Steps

To unblock the Major Triads Lesson 3 benchmark, we recommend a two-layered progression:

1. **Geometry-Safe Layout Refinements**:
   - Refine the relative barline-crossing crossing filters to safely accept compact outer barlines on Lesson 3, Page 1, System 7.
   - Improve system outer boundary heuristics without using pitch-based DP shortcuts.
2. **Timing-Source Hardening**:
   - Supply a clean, structurally aligned MusicXML parent for Lesson 3 to act as a trustworthy timing oracle, bypassing Audiveris OMR limitations.
